# Требования к программному продукту "TechStore"
## Интернет-магазин электроники
**Версия документа:** 1.0  
**Дата:** Январь 2025  
**Статус:** Утверждено
---
## 1. Общие сведения
### 1.1 Назначение продукта
Интернет-магазин "TechStore" предназначен для продажи электроники и гаджетов конечным потребителям. Система должна обеспечивать удобный поиск, выбор и покупку товаров онлайн.
### 1.2 Целевая аудитория
- Физические лица, желающие приобрести электронику
- Возрастная группа: 18-65 лет
- Уровень технической грамотности: базовый и выше
### 1.3 Поддерживаемые браузеры
- Google Chrome (последние 2 версии)
- Mozilla Firefox (последние 2 версии)
- Safari (последние 2 версии)
- Microsoft Edge (последние 2 версии)

## 2. Функциональные требования
### 2.1 Модуль регистрации пользователя
#### REQ-REG-001: Форма регистрации
Система должна предоставлять форму регистрации со следующими полями:
- Имя пользователя (обязательное)
- Email (обязательное)
- Телефон (необязательное)
- Пароль (обязательное)
- Подтверждение пароля (обязательное)
- Чекбокс согласия с условиями использования (обязательное)
#### REQ-REG-002: Валидация имени
- Имя должно содержать минимум 2 символа
- Имя не должно состоять только из пробелов
- Имя не должно содержать цифры и спецсимволы
#### REQ-REG-003: Валидация email
- Email должен соответствовать формату: username@domain.tld
- Должна присутствовать часть до символа @
- Должен присутствовать домен после @
- Должна присутствовать точка и доменная зона (минимум 2 символа)
- Пример валидного email: user@example.com
#### REQ-REG-004: Валидация телефона
- Телефон должен содержать только цифры, пробелы, дефисы и символ +
- Телефон должен начинаться с +7 или 8
- Телефон должен содержать 11 цифр (для России)
- Пример валидного телефона: +7 (999) 123-45-67
#### REQ-REG-005: Валидация пароля
- Минимальная длина пароля: 8 символов
- Максимальная длина пароля: 50 символов
- Пароль должен содержать минимум одну заглавную букву
- Пароль должен содержать минимум одну цифру
- Пароль должен содержать минимум один спецсимвол (!@#$%^&*)
#### REQ-REG-006: Подтверждение пароля
- Значение поля "Подтверждение пароля" должно точно совпадать с полем "Пароль"
#### REQ-REG-007: Согласие с условиями
- Регистрация невозможна без установки чекбокса согласия с условиями испо��ьзования
- Ссылка на условия использования должна открываться в новом окне
#### REQ-REG-008: Успешная регистрация
- После успешной регистрации пользователь должен быть перенаправлен на главную страницу
- Пользователь должен быть автоматически авторизован
- Должно отображаться приветственное сообщение
#### REQ-REG-009: Очистка ошибок валидации
- При исправлении данных в поле, сообщение об ошибке должно автоматически исчезать
- Ошибки должны очищаться при начале ввода в соответствующее поле

### 2.2 Модуль авторизации пользователя
#### REQ-AUTH-001: Форма авторизации
Система должна предоставлять форму входа со следующими полями:
- Email (обязательное)
- Пароль (обязательное)
- Чекбокс "Запомнить меня" (необязательное)
#### REQ-AUTH-002: Валидация при входе
- Email должен соответствовать формату email
- Пароль должен содержать минимум 1 символ
#### REQ-AUTH-003: Успешная авторизация
- После успешного входа пользователь перенаправляется на главную страницу
- В шапке сайта должно отображаться имя пользователя
- Кнопка "Войти" должна смениться на кнопку профиля
#### REQ-AUTH-004: Ссылка "Забыли пароль?"
- Ссылка должна вести на страницу восстановления пароля
- На странице восстановления должна быть форма ввода email
#### REQ-AUTH-005: Авторизация через социальные сети
- Должны быть доступны кнопки входа через Google и VKontakte
- При клике должен открываться popup окно авторизации соответствующего сервиса
#### REQ-AUTH-006: Переход к регистрации
- На странице входа должна быть ссылка "Зарегистрироваться"
- Ссылка должна вести на страницу регистрации
---
### 2.3 Модуль каталога товаров
#### REQ-CAT-001: Отображение списка товаров
Каждый товар в каталоге должен отображать:
- Изображение товара
- Название товара
- Текущую цену
- Старую цену (если есть скидка)
- Размер скидки в процентах (если есть)
- Рейтинг товара (от 0 до 5 звёзд)
- Количество отзывов
- Статус наличия
- Кнопку "В корзину"
#### REQ-CAT-002: Валидация данных товара
- Название товара не должно быть пустым
- Цена должна быть положительным числом больше 0
- Рейтинг должен быть в диапазоне от 0 до 5
- Количество отзывов должно быть неотрицательным числом (≥ 0)
- Старая цена должна быть больше текущей цены
#### REQ-CAT-003: Отображение скидки
- Скидка рассчитывается как: ((старая_цена - новая_цена) / старая_цена) * 100%
- Скидка должна отображаться со знаком минус (например, "-20%")
- Скидка отображается только если старая цена больше текущей
#### REQ-CAT-004: Рейтинг товара
- Рейтинг отображается в виде звёзд
- Количество закрашенных звёзд соответствует рейтингу
- Рейтинг 4.5 отображает 4.5 закрашенных звезды
#### REQ-CAT-005: Статус наличия
- Товар "В наличии" — кнопка "В корзину" активна
- Товар "Нет в наличии" — кнопка "В корзину" неактивна (disabled)
#### REQ-CAT-006: Счётчик товаров
- Под фильтрами должен отображаться счётчик "Показано X товаров"
- X должно соответствовать количеству отображаемых товаров после фильтрации
- Слово "товар" должно склоняться: 1 товар, 2-4 товара, 5+ товаров
---
### 2.4 Модуль поиска
#### REQ-SEARCH-001: Поле поиска
- Поле поиска должно располагаться в шапке сайта
- Должен быть placeholder "Поиск товаров..."
- Должна быть иконка поиска
#### REQ-SEARCH-002: Выполнение поиска
- Поиск должен выполняться при нажатии клавиши Enter
- Поиск должен выполняться при клике на иконку поиска
- Поиск НЕ должен быть чувствителен к регистру
- "Смартфон", "смартфон", "СМАРТФОН" должны давать одинаковый результат
#### REQ-SEARCH-003: Результаты поиска
- Должны отображаться товары, название которых содержит поисковый запрос
- При пустом запросе отображаются все товары
- Счётчик должен показывать количество найденных товаров
#### REQ-SEARCH-004: Очистка поиска
- Кнопка очистки (✕) должна очищать поле ввода
- Кнопка очистки должна сбрасывать результаты поиска
- После очистки должны отображаться все товары
---
### 2.5 Модуль фильтрации и сортировки
#### REQ-FILTER-001: Сортировка по цене
Доступные варианты сортировки:
- "По умолчанию" — исходный порядок товаров
- "Сначала дешёвые" — от минимальной цены к максимальной
- "Сначала дорогие" — от максимальной цены к минимальной
#### REQ-FILTER-002: Фильтр по наличию
- Чекбокс "Только в наличии"
- При активации показываются только товары со статусом "В наличии"
- При деактивации показываются все товары
#### REQ-FILTER-003: Сброс фильтров
- Кнопка "Сбросить фильтры" должна:
  - Сбрасывать сортировку на "По умолчанию"
  - Снимать чекбокс "Только в наличии"
  - Очищать поисковый запрос
  - Показывать все товары
---
### 2.6 Модуль корзины
#### REQ-CART-001: Добавление товара
- При клике на "В корзину" товар добавляется в корзину
- Счётчик в иконке корзины увеличивается
- Если товар уже в корзине, увеличивается его количество
#### REQ-CART-002: Иконка корзины
- В шапке сайта отображается иконка корзины
- Рядом с иконкой отображается количество товаров
- Если корзина пуста, счётчик НЕ отображается
- При клике открывается панель корзины
#### REQ-CART-003: Панель корзины
- Панель выезжает справа
- Отображается затемнённый оверлей
- Клик по оверлею закрывает корзину
- Кнопка "✕" закрывает корзину
#### REQ-CART-004: Содержимое корзины
Для каждого товара отображается:
- Изображение
- Название
- Цена за единицу
- Количество с кнопками +/-
- Кнопка удаления
#### REQ-CART-005: Изменение количества
- Кнопка "+" увеличивает количество на 1
- Кнопка "-" уменьшает количество на 1
- Минимальное количество: 1
- При количестве 1 кнопка "-" должна быть неактивна или удалять товар
#### REQ-CART-006: Расчёт итоговой суммы
- Итого = сумма (цена × количество) для всех товаров
- Сумма должна пересчитываться при изменении количества
- Сумма должна отображаться в формате "XX XXX ₽"
#### REQ-CART-007: Пустая корзина
- Если корзина пуста, отображается сообщение "Корзина пуста"
- Кнопка "Оформить заказ" неактивна при пустой корзине
#### REQ-CART-008: Переход к оформлению
- Кнопка "Оформить заказ" ведёт на страницу оформления
- Кнопка активна только если в корзине есть товары
---
### 2.7 Модуль оформления заказа
#### REQ-ORDER-001: Форма оформления заказа
Обязательные поля:
- ФИО получателя
- Email
- Телефон
- Город
- Адрес доставки (при выборе доставки курьером)
#### REQ-ORDER-002: Валидация ФИО
- Минимум 2 символа
- Не должно состоять только из пробелов
- Должно содержать только буквы и пробелы
#### REQ-ORDER-003: Валидация email
- Должен соответствовать формату email (см. REQ-REG-003)
#### REQ-ORDER-004: Валидация телефона
- Должен соответствовать формату телефона (см. REQ-REG-004)
#### REQ-ORDER-005: Выбор города
- Выпадающий список с городами
- Каждый город должен присутствовать в списке только один раз
- Обязательный выбор
#### REQ-ORDER-006: Способ доставки
Доступные варианты:
- Самовывоз (бесплатно)
- Доставка курьером (300 ₽)
#### REQ-ORDER-007: Адрес доставки
- Поле обязательно при выборе "Доставка курьером"
- Поле необязательно при выборе "Самовывоз"
#### REQ-ORDER-008: Способ оплаты
Доступные варианты (все должны быть активны):
- Картой онлайн
- При получении
Недоступные варианты (должны быть неактивны):
- В кредит (отображается как недоступный)
#### REQ-ORDER-009: Комментарий к заказу
- Необязательное поле
- Максимальная длина: 500 символов
- Отображается счётчик оставшихся символов
#### REQ-ORDER-010: Расчёт стоимости заказа
- Подытог = сумма всех товаров с учётом количества
- Доставка = 0 ₽ (самовывоз) или 300 ₽ (курьер)
- Итого = Подытог + Доставка
#### REQ-ORDER-011: Успешное оформление
- После оформления отображается сообщение об успехе
- Отображается уникальный номер заказа
- Корзина полностью очищается
- Номер заказа должен быть уникальным для каждого заказа
---
### 2.8 Модуль профиля пользователя
#### REQ-PROFILE-001: Информация о пользователе
Отображается:
- Имя пользователя
- Email
- Телефон (если указан)
#### REQ-PROFILE-002: История заказов
Для каждого заказа отображается:
- Номер заказа
- Дата заказа (в формате ДД.ММ.ГГГГ)
- Статус заказа
- Сумма заказа
#### REQ-PROFILE-003: Валидация даты заказа
- Дата должна быть валидной (существующей)
- Дата не должна быть в будущем
- Формат: ДД.ММ.ГГГГ
#### REQ-PROFILE-004: Статусы заказа
Возможные статусы:
- Обрабатывается
- Доставляется
- Доставлен
- Отменён
Статус должен отображаться полностью, без обрезания
#### REQ-PROFILE-005: Сумма заказа
- Сумма должна быть положительным числом
- Отображается в формате "XX XXX ₽"
#### REQ-PROFILE-006: Кнопка "Подробнее"
- При клике открывается детальная информация о заказе
- Отображаются товары, адрес, способ оплаты
#### REQ-PROFILE-007: Настройки уведомлений
- Переключатели должны сохранять состояние
- При включении/выключении изменения сохраняются
#### REQ-PROFILE-008: Удаление аккаунта
- Кнопка "Удалить аккаунт"
- Перед удалением должно появляться окно подтверждения
- Удаление происходит только после подтверждения
#### REQ-PROFILE-009: Выход из аккаунта
- Кнопка "Выйти" завершает сессию
- Пользователь перенаправляется на главную страницу
- В шапке снова отображается кнопка "Войти"

### 2.9 Навигация и общие элементы
#### REQ-NAV-001: Шапка сайта (Header)
Должна содержать:
- Логотип (кликабельный, ведёт на главную)
- Поле поиска
- Навигационные ссылки (Каталог, Акции, Доставка, Контакты)
- Иконку корзины
- Кнопку входа/профиля
- Контактный телефон (валидный формат)
#### REQ-NAV-002: Логотип
- При клике ведёт на главную страницу
- Название магазина должно быть единообразным везде (TechStore)
#### REQ-NAV-003: Навигационные ссылки
- Все ссылки должны быть кликабельными
- Каждая ссылка должна вести на соответствующую страницу
- Активная страница должна быть выделена
#### REQ-NAV-004: Подвал сайта (Footer)
Должен содержать:
- Информацию о компании
- Контактные данные (валидный email, телефон, адрес)
- Ссылки на социальные сети (рабочие)
- Копирайт с актуальным годом
#### REQ-NAV-005: Контактная информация
- Email должен быть валидным (с точкой перед доменной зоной)
- Телефон должен быть в правильном формате
- Адрес должен быть реалистичным (номер дома > 0)
#### REQ-NAV-006: Кнопка "Наверх"
- Появляется при прокрутке страницы вниз
- При клике плавно прокручивает страницу наверх
#### REQ-NAV-007: Орфография и грамматика
- Все тексты должны быть без орфографических ошибок
- Правильное склонение слов
- Правильные окончания

## 3. Нефункциональные требования
### 3.1 Производительность
- Страница должна загружаться менее чем за 3 секунды
- Отклик на действия пользователя — менее 100 мс
### 3.2 Удобство использования
- Интерфейс должен быть интуитивно понятным
- Все интерактивные элементы должны иметь hover-эффект
- Сообщения об ошибках должны быть понятными
### 3.3 Совместимость
- Адаптивный дизайн для мобильных устройств
- Корректное отображение на разрешениях от 320px до 2560px
